## 1. プロトタイプの目的と限界

### 目的
「六つの理（摂理・調和・循環・構造・秩序・和）」という哲学的価値基準を、
AI/AGI/ASI が定量的に自己評価できる形に **構造化・コード化すること**。

### このプロトタイプが行っていること
- 六原理を軸とした評価フレームワークの **アーキテクチャ設計**
- スコア計算・可視化・フェーズ検出・多エージェント評価の **ロジック実装**
- 概念間の関係性（共鳴・遷移・集合調和）の **数理モデル化の試み**

### このプロトタイプが行っていないこと
| 項目 | 状態 |
|---|---|
| 実際の AI システムへの適用・測定 | 未実施 |
| 数値パラメータの実験的最適化 | 未実施 |
| 他の AI 評価フレームワーク（ARC-Eval, HELM, BIG-Bench 等）との比較検証 | 未実施 |
| 人間の評価者による妥当性検証（face validity） | 未実施 |
| 時系列データによるモデルの予測精度評価 | 未実施 |
| 査読・外部レビュー | 未実施 |

- 🟢 **標準手法** — 統計学・情報理論の確立した手法をそのまま適用。ただし問題設定への適合性は未検証。

#### B-2. 共鳴行列（ResonanceMatrix）

```
M = [[1.00, 0.45, 0.30, 0.55, 0.70, 0.35],   # 摂理
     [0.45, 1.00, 0.60, 0.30, 0.25, 0.80],   # 調和
     [0.30, 0.60, 1.00, 0.25, 0.20, 0.50],   # 循環
     [0.55, 0.30, 0.25, 1.00, 0.75, 0.40],   # 構造
     [0.70, 0.25, 0.20, 0.75, 1.00, 0.30],   # 秩序
     [0.35, 0.80, 0.50, 0.40, 0.30, 1.00]]   # 和
```

| 項目 | 状態 | 説明 |
|---|---|---|
| 全オフ対角成分（15 個の値） | 🔴 根拠なし | AI が哲学的解釈で「摂理と秩序は強く共鳴する（0.70）」等を設定。完全に主観的。 |
| 行列の対称性（M = Mᵀ） | 🟡 弱い根拠 | 双方向影響を等価と仮定。実際には非対称な影響関係が存在しうる。 |
| 正定値性の非保証 | ⚠️ 注意 | 現行行列の正定値性を保証するチェックが未実装。スペクトル解析が必要。 |

#### B-3. 共鳴指数の算式

```
R = sᵀ · M · s  /  (‖s‖² · tr(M))
```

| 項目 | 状態 | 説明 |
|---|---|---|
| 二次形式 sᵀMs の採用 | 🟡 弱い根拠 | グラフ信号処理での平滑性指標と類似しているが、本問題への適用は未検証。 |
| tr(M) による正規化 | 🟡 弱い根拠 | [0,1] への値域制約のための正規化。行列ノルム（フロベニウスノルム等）との比較が必要。 |
| 共鳴ボーナス上限 10%（`boost * 0.1`） | 🔴 根拠なし | 「共鳴による実効スコア増幅は最大 10%」という根拠なし。 |

#### B-4. 不協和度の算式

```
dissonance(i, j) = M[i,j] × |score_i - score_j|
```

| 項目 | 状態 | 説明 |
|---|---|---|
| 不協和度の定義式 | 🔴 根拠なし | AI が「共鳴係数 × スコア差 = 改善インパクト」と仮定して定義。 |
| 閾値 0.15 | 🔴 根拠なし | 表示フィルタとして使用。上位 5 件の閾値に科学的根拠なし。 |

#### B-5. メタ評価器（MetaEvaluator）

```python
coverage   = min(1.0, metrics_count / 20.0)   # 20指標で満点
stability  = max(0.0, 1.0 - volatility * 5)   # 揺らぎ係数
meta_score = 0.25×coverage + 0.25×confidence + 0.20×stability
           + 0.20×coherence + 0.10×coverage_ratio
```

| 項目 | 状態 | 説明 |
|---|---|---|
| 20 指標で満点（正規化係数 1/20） | 🔴 根拠なし | 「20 個あれば十分」という根拠なし。 |
| 揺らぎペナルティ係数 5 | 🔴 根拠なし | 標準偏差 0.2 でスタビリティ=0 になる設計だが、0.2 を最悪値とする根拠なし。 |
| 5 次元の重みベクトル | 🔴 根拠なし | 各次元の相対重要度は未検証。 |

```
AGI 閾値: 汎化=0.75, 因果推論=0.78, メタ学習=0.70,
```

| 項目 | 状態 | 説明 |
|---|---|---|

| 7 次元すべて閾値超過で遷移（AND 条件） | 🟡 弱い根拠 | 最弱次元がボトルネックになる設計。OR 条件や加重和での遷移判定も同等に妥当。 |
| 能力次元の選択（7 次元） | 🔴 根拠なし | AI が「AGI に必要」と推測して選定。ARC-AGI, MMLU 等との対応関係が未検証。 |

#### C-2. 遷移確率のシグモイド式

```python
P = 1 / (1 + exp(total_deficit × n × 3.0))
```

| 項目 | 状態 | 説明 |
|---|---|---|
| 急峻パラメータ 3.0 | 🔴 根拠なし | シグモイドの傾きを制御する。この値が変わると確率が大きく変化する。 |
| n（次元数=7）を係数に含む | 🟡 弱い根拠 | 次元数で総不足量を正規化する意図だが、個別不足の最大値で代替すべき可能性がある。 |
| シグモイド関数の選択自体 | 🟡 弱い根拠 | 遷移確率が累積正規分布・ステップ関数の方が適切な可能性がある。 |

#### C-3. Lyapunov 臨界性指数の近似

```python
lyap = mean(log(|Δs_{n+1} / Δs_n|))      # 有限差分比の対数平均
criticality = clip((lyap + 3.0) / 6.0, 0, 1)   # [-3, +3] → [0, 1]
```

| 項目 | 状態 | 説明 |
|---|---|---|
| 有限差分比による Lyapunov 近似 | 🟡 弱い根拠 | 本来の Lyapunov 指数は連続軌道の分離率。スカラー時系列への適用は厳密でない。 |
| 正規化範囲 [-3, +3] | 🔴 根拠なし | スコア時系列の有限差分比の対数が [-3, +3] 内に収まるという保証がない。 |
| 臨界性 = Lyapunov 指数という同一視 | 🔴 根拠なし | カオス縁（edge of chaos）と Lyapunov 指数の関係は非線形力学系の特定設定での議論。AW スコア時系列への適用は仮設的。 |

## 3. 横断的な仮定条件

以下の仮定はすべてのモジュールに影響する根本的な未検証前提です。

| # | 仮定 | 影響範囲 | 検証可能な反論 |
|---|---|---|---|
| H-1 | 六つの理は AI 知性を評価するのに必要十分な6次元を構成する | 全モジュール | 他の枠組み（倫理的 AI、フェアネス、解釈可能性等）が欠落している可能性 |
| H-2 | 各原理スコアは [0,1] の実数で線形に表現できる | 全モジュール | 質的・離散的・非線形な尺度が適切な可能性 |
| H-3 | 原理スコアの加重和が「知恵（Wisdom）」を表す | 全モジュール | Wisdom は単純な線形結合で捉えられない非線形・創発的概念の可能性 |
| H-4 | AI システムは自分自身を正確にスコアリングできる（自己評価の精度） | 全モジュール | 自己評価バイアス（Dunning-Kruger 効果等）の可能性 |

| H-7 | 集合的 Wa は個別スコアの統計量で計算できる | aw_collective_harmony.py | 集合知は創発的であり、個体のスコアには還元できない可能性 |

## 4. 検証課題リスト（Research Agenda）

### 🔬 即座に取り組み可能な検証

1. **感度分析（Sensitivity Analysis）**
   - 共鳴行列 M の各成分を ±0.2 変化させたとき、AW スコアがどの程度変動するか
   - 原理重みベクトルを一様分布に変えたとき、ランキングが変化するか
   - `python`: `scipy.stats.sobol_indices` で実施可能

2. **内部一貫性の検証**
   - 22 個の評価メトリクスが 6 原理を重複なく・漏れなく測定しているか（CFA: 確認的因子分析）
   - クロンバックα係数で各原理のメトリクス群の内的整合性を測定
   - `python`: `pingouin.cronbach_alpha`

3. **共鳴行列の正定値性確認**
   ```python
   import numpy as np
   eigenvalues = np.linalg.eigvals(M)
   assert (eigenvalues > 0).all(), "ResonanceMatrix is not positive definite"
   ```

4. **Lyapunov 近似の精度評価**
   - 既知の Lyapunov 指数を持つ人工時系列（ロジスティック写像等）でテスト
   - 真値との誤差を定量化

### 📊 中期的に必要な検証

5. **外部妥当性検証（External Validity）**
   - GPT-4, Gemini Ultra, Llama 3 等の公開モデルに本フレームワークを適用
   - HumanEval / MMLU / BIG-Bench での客観スコアとの相関係数を計算
   - 期待: 「摂理スコア ↔ 論理推論ベンチマーク」に正の相関が出るはず

6. **フェーズ閾値の経験的推定**
   - 複数の AI 研究者に「これは AGI か？」とラベルを付けてもらい
   - 多数決ラベルと本モデルの分類が一致するよう閾値を回帰推定

7. **多エージェント評価の現実適用**
   - 実際の LLM-as-a-Judge マルチエージェント設定で Collective Wa スコアを測定
   - 集合的 Wa が高いグループの意思決定品質が高いかを検証

### 🔭 長期的・根本的な検証

8. **「六つの理」の普遍性の検証**
   - 異なる文化的・哲学的枠組み（儒教の五常、Aristotle の徳倫理学等）との対応マッピング
   - 六つの理で捉えられない AI の問題事例の収集

9. **Wisdom の定義問題**
   - 心理学的 wisdom 研究（Sternberg, Ardelt 等）との比較
   - 「AW スコアが高いシステムの出力を人間は賢明と評価するか」の実験

```bash
# 各モジュールのデモ実行（ハードコードされた仮想スコアで動作）
```

### 入力データ形式

全モジュールは `float` のスコア辞書を入力として受け取る。  
外部データをつなぐには `engine.bulk_update(dict)` を使用。

```python
# 実システムのスコアを外部から注入する例
from aw_framework_v2 import AWEvaluationEngine, PrincipleType, initialize_default_metrics

engine = AWEvaluationEngine(agent_id="GPT-4-eval")
initialize_default_metrics(engine)

# 外部ベンチマーク結果を正規化して代入
engine.bulk_update({
    "logical_consistency": normalize(mmlu_score),   # MMLU → [0,1]
    "sustainability":      0.0,                      # 現状未測定
    # ...
})
```

## 6. 用語定義

| 用語 | このコードでの定義 | 一般的な定義との差異 |
|---|---|---|
